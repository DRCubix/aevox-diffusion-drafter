# 01 — Method

## 1. The drafter: a tri-mode diffusion LM
`Nemotron-Labs-Diffusion-3B` is a standard transformer (Ministral-based) trained with a **joint AR + diffusion** objective. It runs in three modes from one set of weights by switching attention patterns:
- **AR** — causal, sequential.
- **Diffusion** — block-masked, parallel denoising.
- **Self-speculation** — its own diffusion head drafts, its own AR head verifies.

NVIDIA themselves ship a `linear_spec_lora` adapter that sharpens the *diffusion drafter* for self-speculation. **This project applies the same LoRA technique, but retargets the drafter to a *different* model — the 120B.**

## 2. Cross-model speculative decoding
At `nfe=1` (single denoising step), the drafter:
1. Takes the accepted prefix (causal KV cache) + a block of `K` `<mask>` tokens, seeded with the last accepted token.
2. Runs **one bidirectional diffusion forward** → fills all `K` positions (argmax of the diffusion head).
3. The 120B verifies the block in **one forward**, accepting the longest prefix that matches its own next tokens, plus one bonus token.

Because the tokenizer is byte-identical, the 120B can verify the 3B's token IDs directly. Correctness (loss-lessness) is guaranteed by the verifier regardless of draft quality; draft quality only affects **how many** tokens are accepted.

## 3. Alignment training (the core contribution)
The unaligned 3B predicts the 120B's tokens only moderately well (accept 2.26). We **LoRA-align** it by continued training on the 120B's own outputs, using the model's native bidirectional diffusion objective:

- **Data:** `(prompt, 120B-completion)` token pairs (10K, code-heavy), generated by the 120B at temp=1.0 / full-reasoning (its recommended setting).
- **Objective:** `model(input_ids, labels=input_ids, loss_mask)` → LLaDA-style cross-entropy on **masked completion tokens only** (`loss_mask=1` on completion, `0` on prompt). The drafter learns to denoise/predict the 120B's continuation.
- **LoRA:** rank 16 on `q/k/v/o + gate/up/down` (24.7M trainable, 0.64%); embedding and the diffusion head stay frozen.
- **Stability:** `eps=0.05` floors the `1/p_mask` loss weight (prevents NaN spikes), non-finite batches are skipped, `lr=5e-5`.
- **Mode discipline:** training resets the per-layer `diffusion_lm=True` each epoch — the eval toggles it to causal, and leaving it flipped silently trains the wrong objective (see [06-gotchas.md](06-gotchas.md)).

## 4. Evaluation
`eval_lora_acceptance.py` measures **accepted tokens per target forward** by simulating the spec-decode loop against the 120B's exact token sequence (the byte-identical tokenizer makes IDs comparable). This is the standard speculative-decoding acceptance metric — directly comparable to MTP's reported 2.75. We do not need to run the 120B during eval; the saved completions are the verification target.

## 5. From acceptance to speedup
Spec-decode speedup vs AR ≈ `accept / (1 + nfe·r)`, where `r = (drafter forward time)/(target forward time)`. Measured: drafter ≈ 34 ms (bf16), target ≈ 62 ms → `r≈0.55`. Quantizing the drafter to **NVFP4** drops `r≈0.11`, which is what turns the 2.79 acceptance into a projected **~2.5×** (see [02-results.md](02-results.md)).

## 6. What we also tried (and why it didn't help)
- **Logit-KL distillation** (`train_kl.py`): distill on the 120B's top-k logits instead of hard tokens. Did **not** beat hard-token CE (2.17 ≤ 2.30) because the 120B's per-token distribution is extremely **peaked** → the soft signal collapses to the hard token.
- **Multi-step diffusion** (`nfe_sweep.py`): higher `threshold` raises acceptance marginally (2.30→2.57) but `nfe` explodes (1→13), cratering the speedup.
